# Bytewise Design System Guidelines

## Brand Identity

### Typography
- Primary font: **League Spartan** (headings, brand elements)
- Secondary font: **Work Sans** (subheadings, labels, buttons)
- Body font: **Quicksand** (body text, inputs)

### Color Palette
- **Primary Background**: Pastel Yellow (#fef7cd)
- **Primary Accent**: Pastel Blue (#a8dadc)
- **Text**: Black (#1d1d1b) or White (#ffffff)
- **Secondary**: Light grays for cards and backgrounds

### Logo Standards

#### CSS-Based Logo Implementation
The Bytewise logo is implemented using pure CSS for optimal performance and scalability:

**Main Logo Text**: "bytewise"
- Font: League Spartan, 700 weight
- Color: Pastel Blue (#a8dadc)
- Case: Lowercase
- Letter spacing: -0.02em (slight tightening)

**Tagline**: "Nutritionist"
- Font: League Spartan, 400 weight
- Color: Black (#1d1d1b)
- Letter spacing: 0.15em (elegant spacing)
- Alignment: 'N' character positioned directly under the 't' of "bytewise"

#### Logo Sizing System
```css
/* Small (Header/Navigation) */
.bytewise-logo-sm .bytewise-logo-main { font-size: 1.5rem; /* 24px */ }
.bytewise-logo-sm .bytewise-logo-tagline { 
  font-size: 0.75rem; /* 12px */
  margin-top: 0.125rem; /* 2px */
  text-indent: 1.8ch; /* Character-based alignment */
}

/* Medium (Standard) */
.bytewise-logo-md .bytewise-logo-main { font-size: 2.25rem; /* 36px */ }
.bytewise-logo-md .bytewise-logo-tagline { 
  font-size: 1rem; /* 16px */
  margin-top: 0.25rem; /* 4px */
  text-indent: 2.0ch; /* Character-based alignment */
}

/* Large (Landing/Authentication) */
.bytewise-logo-lg .bytewise-logo-main { font-size: 3rem; /* 48px */ }
.bytewise-logo-lg .bytewise-logo-tagline { 
  font-size: 1.25rem; /* 20px */
  margin-top: 0.375rem; /* 6px */
  text-indent: 2.2ch; /* Character-based alignment */
}
```

#### Character-Based Alignment System
The tagline uses CSS `text-indent` with character units (ch) for precise alignment:
- **1ch = width of one character** in the current font
- Ensures 'N' of "Nutritionist" aligns perfectly under 't' of "bytewise"
- Scales proportionally with font size changes
- Maintains alignment across different screen resolutions

#### Logo Component Structure
```jsx
// Usage in React components
<LogoBrand size="sm" />  // Header/Navigation
<LogoBrand size="md" />  // Standard usage
<LogoBrand size="lg" />  // Landing/Auth pages

// Implementation structure
<div className="bytewise-logo">
  <div className="bytewise-logo-main">bytewise</div>
  <div className="bytewise-logo-tagline">Nutritionist</div>
</div>
```

#### Accessibility Features
- **High Contrast Support**: Automatic color adjustments for accessibility
- **Reduced Motion**: Hover animations disabled for users who prefer reduced motion
- **Print Optimization**: Special print styles for documents
- **Dark Theme Support**: Maintains pastel blue for logo, adjusts tagline to white
- **Screen Reader Friendly**: Proper semantic structure

#### Usage Guidelines
1. **Minimum Size**: Never scale below small (sm) size specifications
2. **Clear Space**: Maintain minimum padding equivalent to the height of the 'N' character
3. **Background Contrast**: Ensure sufficient contrast for both logo and tagline colors
4. **Interactive States**: Subtle hover scale (1.02x) for clickable instances
5. **Loading States**: Optional fade-in animation for page loads

#### Technical Implementation
- **Zero External Dependencies**: Pure CSS implementation
- **Performance Optimized**: No image assets, uses web fonts
- **Scalable**: Vector-based text scales perfectly at any resolution
- **Customizable**: Easy color and size modifications via CSS variables
- **Cross-Browser Compatible**: Supports all modern browsers

## Hero Component Standard

### Structure
All hero components across the app must follow this consistent pattern with **seamless header integration**:

#### 1. Container Structure
```jsx
{/* Hero Section with Large Photo Overlay - Seamless with Header */}
<div className="relative -mx-3 mb-6">
  <div className="h-64 relative overflow-hidden">
    <ImageWithFallback
      src="[contextual-image-url]"
      alt="[descriptive-alt-text]"
      className="absolute inset-0 w-full h-full object-cover"
    />
    <div className="absolute inset-0 bg-gradient-to-t from-black/80 via-black/40 to-transparent" />
    
    {/* Text Overlay Content - Positioned below header */}
    <div className="absolute inset-x-4 bottom-6 top-16 flex flex-col justify-end text-white">
      {/* Content structure here */}
    </div>
  </div>
</div>
```

#### **Seamless Header-Hero Integration**
- **No spacing between header and hero**: Hero components start immediately under the fixed header
- **Main content has no top padding**: Allows hero to extend to the very top of the content area
- **Header overlay effect**: The fixed header appears to float over the hero background
- **Visual continuity**: Creates an immersive, full-screen experience

#### 2. Required Elements
- **Full-width container**: Use `relative -mx-3` (or `-mx-4` for dashboard) to extend beyond normal content padding
- **Fixed height**: `h-64` for consistent hero height across all pages
- **Background image**: High-quality, contextual image with proper object-cover
- **Gradient overlay**: `bg-gradient-to-t from-black/80 via-black/40 to-transparent` for text readability
- **Text overlay**: Positioned at `bottom-6 left-4 right-4` with white text

#### 3. Content Layout
The hero content must follow this three-column layout:

**Left Column (Title & Description):**
```jsx
<div>
  <p className="text-sm opacity-90 mb-1" style={{ fontFamily: "'Quicksand', sans-serif", fontSize: "0.875rem", fontWeight: 400 }}>[Context/Subtitle]</p>
  <h1 style={{ fontFamily: "'League Spartan', sans-serif", fontSize: "1.875rem", fontWeight: 700, lineHeight: 1.2 }}>[Page Title]</h1>
  <p className="text-sm opacity-90 mt-1" style={{ fontFamily: "'Quicksand', sans-serif", fontSize: "0.875rem", fontWeight: 400 }}>[Description]</p>
</div>
```

**Right Column (Statistics Card):**
```jsx
<div className="text-center bg-white/20 backdrop-blur-sm rounded-2xl px-4 py-3">
  <div className="flex items-center space-x-2 mb-1">
    <[Icon] className="text-[theme-color]" size={16} />
    <span className="text-2xl font-bold" style={{ fontFamily: "'League Spartan', sans-serif", fontSize: "1.5rem", fontWeight: 700 }}>[Number]</span>
  </div>
  <p className="text-xs opacity-90" style={{ fontFamily: "'Quicksand', sans-serif", fontSize: "0.75rem", fontWeight: 400 }}>[Label]</p>
</div>
```

**Center (Progress Ring):**
```jsx
<div className="relative w-20 h-20">
  <svg className="w-20 h-20 transform -rotate-90" viewBox="0 0 100 100">
    <circle cx="50" cy="50" r="35" stroke="rgba(255,255,255,0.3)" strokeWidth="6" fill="transparent" />
    <circle cx="50" cy="50" r="35" stroke="[theme-color]" strokeWidth="6" fill="transparent" 
      strokeDasharray={`${2 * Math.PI * 35}`}
      strokeDashoffset={`${2 * Math.PI * 35 * (1 - progressPercentage)}`}
      strokeLinecap="round" />
  </svg>
  <div className="absolute inset-0 flex items-center justify-center">
    <div className="text-center">
      <div className="text-lg font-bold" style={{ fontFamily: "'League Spartan', sans-serif", fontSize: "1.125rem", fontWeight: 700 }}>[Percentage]%</div>
      <div className="text-xs opacity-75" style={{ fontFamily: "'Quicksand', sans-serif", fontSize: "0.75rem", fontWeight: 400 }}>[Label]</div>
    </div>
  </div>
</div>
```

#### 4. Image Selection Guidelines
- **Dashboard**: General healthy food/ingredients lifestyle images
- **Recipe Builder**: Cooking, ingredients, kitchen utensils
- **Meal Planner**: Meal prep, organized foods, planning setup
- **Calendar**: Progress tracking, analytics, achievement themes
- **Profile**: Personal wellness, user journey, settings themes

#### 5. Responsive Behavior
- Use `-mx-3` for most components (mobile-first)
- Use `-mx-4` for dashboard (slight variation)
- Ensure images work well at mobile dimensions
- Text should remain readable at all screen sizes

#### 6. Accessibility Requirements
- All images must have descriptive alt text
- Proper color contrast maintained with gradient overlay
- Interactive elements must have appropriate ARIA labels
- Text hierarchy properly structured with semantic HTML

#### 7. Header-Hero Integration Requirements
- **Zero spacing**: No padding or margin between fixed header and hero components
- **Immediate visual connection**: Hero images extend to the top edge of the content area
- **Overlay effect**: Fixed header floats above hero background creating depth
- **Consistent implementation**: All pages must follow this seamless pattern
- **Performance**: Smooth scrolling and transitions maintained with seamless design

## Typography Enforcement

### Explicit Font Family Declarations
To override any default component styling, all text elements must include explicit font family declarations:

```jsx
// Headings (League Spartan)
<h1 style={{ fontFamily: "'League Spartan', sans-serif", fontSize: "1.875rem", fontWeight: 700, lineHeight: 1.2 }}>
  Page Title
</h1>

<h2 style={{ fontFamily: "'League Spartan', sans-serif", fontSize: "1.5rem", fontWeight: 600, lineHeight: 1.3 }}>
  Section Title
</h2>

// Subheadings and Labels (Work Sans)
<h3 style={{ fontFamily: "'Work Sans', sans-serif", fontSize: "1.25rem", fontWeight: 600, lineHeight: 1.4 }}>
  Subsection
</h3>

<label style={{ fontFamily: "'Work Sans', sans-serif", fontSize: "0.875rem", fontWeight: 500 }}>
  Form Label
</label>

// Body Text (Quicksand)
<p style={{ fontFamily: "'Quicksand', sans-serif", fontSize: "1rem", fontWeight: 400, lineHeight: 1.6 }}>
  Body content text
</p>

// Buttons (Work Sans)
<button style={{ fontFamily: "'Work Sans', sans-serif", fontSize: "1rem", fontWeight: 500 }}>
  Button Text
</button>
```

### CSS Override Utilities
Use these utility classes when inline styles aren't practical:

```jsx
<h1 className="text-brand-heading">Heading Text</h1>
<h3 className="text-brand-subheading">Subheading Text</h3>
<label className="text-brand-label">Label Text</label>
<p className="text-brand-body">Body Text</p>
<button className="text-brand-button">Button Text</button>
```

## Design Principles

### Brand Logo Consistency
- **Always use LogoBrand component**: Never recreate logo manually
- **Proper sizing**: Use sm/md/lg variants appropriately
- **Clickable in headers**: Logo should navigate to dashboard
- **Semantic structure**: Proper heading hierarchy when logo is primary brand element
- **Accessibility**: Include proper ARIA labels for interactive logo instances

### Seamless Integration Philosophy
- **Zero-gap design**: No spacing between header and hero components
- **Immersive experience**: Hero backgrounds extend to screen edges
- **Header overlay**: Fixed header floats above content for depth
- **Visual continuity**: Seamless flow from navigation to content

### Mobile-First Approach
- All hero components designed for mobile screens first
- Ensure touch targets are minimum 44px for accessibility
- Use appropriate spacing and typography scales
- Seamless header integration optimized for mobile viewports

### Visual Hierarchy
- **Logo prominence**: Brand logo is the primary identifier
- **H1 titles**: `text-3xl font-bold` (League Spartan) with explicit font family
- **Subtitles**: `text-sm opacity-90` (Quicksand) with explicit font family
- **Statistics**: `text-2xl font-bold` for numbers, `text-xs opacity-90` for labels
- **Header elements maintain visibility over hero backgrounds**

### Consistency Requirements
- **Mandatory seamless integration**: All pages must have zero spacing between header and hero
- **Logo standardization**: Use LogoBrand component consistently
- **Typography enforcement**: Always include explicit font family declarations
- Maintain consistent spacing, typography, and layout within hero content
- Use contextually appropriate images and icons
- Ensure proper contrast and readability with header overlay
- Fixed header transparency and backdrop blur for visual depth

### Performance Considerations
- Use ImageWithFallback component for all hero images
- CSS-based logo for optimal performance (no image assets)
- Optimize images for web (800x600 recommended)
- Implement proper loading states and fallbacks
